首先，我们创建一个函数来基于用户是否具有连接性来操作DOM。我们将为它传递一个boolean值。

function hasNetwork(online) {
  const element = document.querySelector(".status");
  // Update the DOM to reflect the current status
  if (online) {
    element.classList.remove("offline");
    element.classList.add("online");
    element.innerText = "Online";
  } else {
    element.classList.remove("online");
    element.classList.add("offline");
    element.innerText = "Offline";
  }
}

接下来是有趣的部分。

navigator.online最初加载页面时，我们将首先用于检查用户是否有互联网。在navigator.online已经支持所有的IE9的，并返回一个boolean值

浏览器以不同的方式实现此功能，有关更多详细信息，请在此处 (opens new window)查看MDN文档。

提示

在Chrome和Safari中，如果浏览器无法连接到局域网（LAN）或路由器，则表明该浏览器处于离线状态；所有其他条件都返回true。因此，尽管您可以假定浏览器在返回假值时处于脱机状态，但不能假设真值必然意味着浏览器可以访问互联网。

// Listen for the page to be finished loading
window.addEventListener("load", () => {
  hasNetwork(navigator.onLine);

注意📝

乍一看，这似乎是没有意义的一步，但是使用Web开发工具和最佳实践的当前状态，您会发现用户经常缓存网页，并且仍然可以脱机访问您的网站。因此，如果他们启动您的应用程序并且可以知道其内容可能已过时，则此初始检查非常有用。

接下来，我们将为联机和脱机事件添加事件侦听器，以便我们可以检查用户的网络状态何时更改。

我们关注两个事件，那就是online和offline。我敢肯定，您会猜到，当用户在离线和在线之间切换时会触发这些事件。


window.addEventListener("load", () => {
  hasNetwork(navigator.onLine);

  window.addEventListener("online", () => {
    // Set hasNetwork to online when they change to online.
    hasNetwork(true);
  });

  window.addEventListener("offline", () => {
    // Set hasNetwork to offline when they change to offline.
    hasNetwork(false);
  });
});

现在，当我们进入应用程序时，当用户更改其网络状态时，我们将获得实时反馈。

那么什么时候使用这个呢？ 也许您需要告诉用户他们由于脱机而无法上传文件，他们看到的实时数据可能不再是最新的，因为他们脱机或其他任何原因。您可能需要出于多种原因来检测网络状态，并且随着Web开发人员的努力，我们尝试使每个人的网络都变得更加用户友好。
